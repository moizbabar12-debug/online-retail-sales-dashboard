# Online Retail Sales Analysis and Interactive Dashboard

An interactive Python dashboard that analyses online retail sales, customer behaviour and revenue trends. The project cleans a year of raw transaction data, explores it with a set of clear charts, and presents an interactive dashboard that a non-technical user can explore through simple filters.

## Project overview

The data comes from a UK-based online gift wholesaler and covers invoice-level transactions: product codes and descriptions, quantities, unit prices, timestamps, customer IDs and countries. The notebook walks through three stages:

1. Cleaning the raw data so the numbers can be trusted.
2. Exploring the data with static charts.
3. Building an interactive dashboard where the charts respond to filters.

## Main objectives

- Clean messy transaction data (duplicates, returns, missing values, extreme orders).
- Show how revenue changes over time and which periods are busiest.
- Identify the products and countries that drive the most revenue.
- Show the distribution of order values and how customer spend relates to order count.
- Present all of this in a dashboard that updates as the user changes filters.

## Dataset description

| Field | Meaning |
|---|---|
| InvoiceNo | Invoice number for each transaction |
| StockCode | Product code |
| Description | Product name |
| Quantity | Units sold on that line |
| InvoiceDate | Date and time of the transaction |
| UnitPrice | Price per unit |
| CustomerID | Customer identifier |
| Country | Customer country |

The dashboard adds a few derived fields during processing, including `LineTotal` (Quantity × UnitPrice) and an order-level table summarising each invoice.

> **Note:** The dataset file `Online Retail.xlsx` is required to run the project. Place it in the project folder before launching. It is not generated by the code.

## Technologies used

- **Python** for all data work
- **pandas** and **NumPy** for cleaning and aggregation
- **Matplotlib** for the charts
- **Plotly** (imported for charting support)
- **ipywidgets** for the interactive filters
- **Voila** to turn the notebook into a web dashboard
- **openpyxl** to read the Excel file

## Interactive dashboard features

The dashboard section provides three controls:

- **Period** — view a single month or all months.
- **Country** — view one country or all countries.
- **Min Order** — hide orders below a chosen value.

Changing any control refreshes all six charts together.

## Main analysis areas

1. Revenue over time
2. Top products by revenue
3. Revenue by country
4. Distribution of order values
5. Customer orders versus total spend
6. Correlation between order value, number of lines and total quantity

## Run the project locally

From inside the project folder:

```
python -m pip install -r requirements.txt
voila online-retail-dashboard.ipynb
```

Voila opens the dashboard in your browser with the code hidden. The filters work because Voila keeps a live Python session running behind the page.

If you just want to view or edit the notebook normally first:

```
python -m pip install -r requirements.txt
jupyter notebook online-retail-dashboard.ipynb
```

Run the cells from top to bottom before relying on the interactive section, since the later cells depend on the earlier ones.

## Troubleshooting

- **`FileNotFoundError: Online Retail.xlsx`** — the dataset is not in the folder, or the filename is different. Make sure the file sits next to the notebook and is named exactly `Online Retail.xlsx`.
- **`Missing optional dependency 'openpyxl'`** — run `python -m pip install openpyxl`, or reinstall from `requirements.txt`.
- **Filters or charts do not appear in Voila** — make sure you launched with `voila online-retail-dashboard.ipynb` and that `ipywidgets` is installed. Refresh the page if the first load is slow.
- **Charts look empty after filtering** — your filters may exclude everything (for example a high Min Order plus a small country). Widen the filters; the dashboard shows a short message when nothing matches.
- **Wrong file path errors on Windows** — the notebook uses a relative path on purpose. Keep the dataset in the same folder rather than pointing at a personal Desktop path.
